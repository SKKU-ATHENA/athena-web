# ATHENA 학습 허브 — QA 시나리오

이 문서는 웹사이트 변경 시 항상 체크하는 품질 검증 시나리오다.

---

## 시나리오 1: 초보자 진입 (First Contact)

**페르소나**: CS 비전공, Python 경험 없음, "RAG"를 처음 들어봄

| 체크포인트 | 기준 |
|-----------|------|
| 홈페이지 첫인상 | 3초 안에 "이 사이트가 뭘 하는 곳인지" 파악 가능한가? |
| 학습 순서 | 어디서부터 시작해야 하는지 명확한가? (환경 세팅 → 임베딩 → ...) |
| 기술 용어 | 전문 용어(ANN, 토큰, 임베딩, 벡터)에 최소 한 줄 설명이 있는가? |
| 코드 블록 | 코드를 복사해서 바로 실행 가능한가? import부터 출력까지 완전한가? |
| 다음 단계 | 각 페이지 끝에서 "다음에 뭘 읽어야 하나?"가 안내되는가? |

---

## 시나리오 2: 시각 일관성 (Visual Consistency)

**검증 방법**: 모든 페이지를 다크/라이트 모드로 각각 확인

| 체크포인트 | 기준 |
|-----------|------|
| FORGE 테마 통일 | 하드코드된 색상(#22b8cf 등) 없이 CSS 변수만 사용하는가? |
| 다크 모드 | 코드 블록, 테이블, 인용문, 배지가 모두 어두운 배경에서 가독적인가? |
| 라이트 모드 | 위와 동일한 요소들이 밝은 배경에서도 명확한가? |
| 그래디언트/글로우 | accent glow가 양쪽 모드에서 자연스러운가? |
| 선택(Selection) | 텍스트 드래그 시 배경과 글자가 모두 보이는가? |
| 스크롤바 | 양쪽 모드에서 스크롤바가 보이는가? |

---

## 시나리오 3: 테이블/코드 가독성 (Data Readability)

**검증 방법**: 데스크톱(1440px)과 모바일(375px) 둘 다 확인

| 체크포인트 | 기준 |
|-----------|------|
| 테이블 가로 스크롤 | 넓은 테이블이 overflow-x-auto로 감싸져 있는가? |
| 테이블 헤더 | 헤더 행이 본문과 시각적으로 구분되는가? |
| 테이블 줄무늬 | 짝수/홀수 행이 구분되는가? |
| 코드 블록 | 긴 줄이 가로 스크롤 가능한가? (줄바꿈 강제 아님) |
| 코드 폰트 | JetBrains Mono가 적용되어 정렬이 맞는가? |
| 인라인 코드 | 백틱 코드가 본문과 시각적으로 구분되는가? (accent glow border) |
| ASCII 다이어그램 | 파이프라인 도표가 폰트 깨짐 없이 정렬되는가? |

## 시나리오 4: 모바일 반응형 (Mobile)

**검증 방법**: 375px 뷰포트

| 체크포인트 | 기준 |
|-----------|------|
| 사이드바 | 모바일에서 Sheet로 전환되는가? |
| 카드 그리드 | 1열로 쌓이는가? |
| 사전과제 스텝 카드 | 좌측 마진(ml-12)이 모바일에서 줄어드는가? |
| 버튼 터치 영역 | 최소 44px 높이인가? |
| 텍스트 크기 | 본문이 최소 14px인가? |
| 테이블 | 가로 스크롤 가능하고 스크롤 힌트가 있는가? |

## 시나리오 5: 콘텐츠 완성도 (Content Quality)

**검증 방법**: 각 마크다운 파일을 처음부터 끝까지 읽기

| 체크포인트 | 기준 |
|-----------|------|
| 출처 검증 | 모든 source가 GitHub 1k+ stars / 공식 문서 / 논문인가? |
| 링크 유효성 | frontmatter의 URL이 모두 유효한가? |
| 코드 실행 가능 | 모든 코드 블록이 copy-paste로 실행 가능한가? |
| 설치 검증 | 각 설치 단계 후 확인 명령어가 있는가? |
| 가격 정보 | API 가격이 최신인가? (분기마다 확인) |
| 다음 단계 연결 | 각 페이지에서 관련 학습 자료로 연결되는가? |

---

## 시나리오 6: 사전과제 플로우 (Pre-assignment Flow)

**페르소나**: 팀원 (CS 전공이지만 RAG 처음)

| 체크포인트 | 기준 |
|-----------|------|
| 전체 흐름 | Part 1 → Part 2 → Part 3 순서가 직관적인가? |
| 각 Step 자기완결 | 각 Step에서 "무엇을", "왜", "어떻게"가 모두 설명되는가? |
| 체감 실험 | 실험 결과가 어떻게 나와야 하는지 예상 결과가 명시되는가? |
| 산출물 명확성 | 각 산출물의 형식, 분량, 제출 방법이 명확한가? |
| 학습 자료 링크 | 관련 학습 자료로의 연결이 자연스러운가? |
| 소요 시간 | 대략적 소요 시간이 명시되는가? |
