## 1. 개요: ATHENA 팀원에게 AI/ML 기초가 왜 필요한가?

ATHENA는 LLM(대규모 언어 모델)을 핵심 엔진으로 사용하는 지식 관리 시스템이다. 직접 모델을 만들 일은 없지만, 다음을 이해해야 프로젝트를 제대로 진행할 수 있다.

- **LLM이 왜 가끔 틀린 답을 하는지** (환각 현상의 원인)
- **RAG가 왜 필요한지** (모델의 한계를 보완하는 이유)
- **모델 선택 시 무엇을 기준으로 판단하는지** (크기, 학습 데이터, 추론 속도)
- **팀 논의에서 "학습", "추론", "파인튜닝" 같은 용어가 나올 때** 무슨 뜻인지

이 문서를 읽고 나면, LLM을 "마법의 블랙박스"가 아니라 "특정 방식으로 훈련된 도구"로 바라볼 수 있게 된다.

## 2. 핵심 개념

### 2.1 모델이란?

#### 비유: 요리 레시피

AI에서 **모델**이란, 입력을 받아서 출력을 내놓는 **규칙 체계**다. 요리에 비유하면 이렇다.

```
재료 (입력)  →  레시피 (모델)  →  요리 (출력)
밀가루, 계란    "반죽 → 굽기"     팬케이크
```

- **재료** = 데이터 (텍스트, 이미지, 숫자 등)
- **레시피** = 모델 (입력을 출력으로 바꾸는 규칙)
- **요리** = 결과 (분류, 예측, 생성된 텍스트 등)

중요한 점: 레시피는 **처음부터 완벽하지 않다**. 여러 번 시행착오를 거쳐 조리법을 개선하듯, AI 모델도 데이터를 보면서 규칙을 **점진적으로 개선**한다. 이 과정이 바로 "학습"이다.

#### 구체적 예시

- 스팸 메일 분류 모델: 이메일(입력) → "스팸" 또는 "정상"(출력)
- 번역 모델: 한국어 문장(입력) → 영어 문장(출력)
- ChatGPT: 사용자 질문(입력) → 답변 텍스트(출력)

이 모든 것이 "모델"이다. 복잡도의 차이가 있을 뿐, 본질은 같다 — **입력을 받아서 유용한 출력을 내놓는 규칙 체계**.

### 2.2 학습 vs 추론

#### 비유: 공부 vs 시험

AI 모델의 생애주기는 딱 두 단계로 나뉜다.

| | 학습 (Training) | 추론 (Inference) |
|---|---|---|
| **비유** | 학생이 교과서로 공부하는 과정 | 시험장에서 문제를 푸는 과정 |
| **무엇을 하는가** | 대량의 데이터를 보면서 패턴을 파악한다 | 새로운 입력을 받아 답을 내놓는다 |
| **언제 일어나는가** | 모델을 만들 때 (보통 사전에, 오래 걸림) | 모델을 사용할 때 (실시간) |
| **비용** | 매우 높음 (GPU 수천 개, 수주~수개월) | 상대적으로 낮음 (질문 1개당 비용) |
| **누가 하는가** | OpenAI, Meta, Google 같은 회사 | 우리 (ATHENA 프로젝트) |

**핵심 포인트**: ATHENA 프로젝트에서 우리가 하는 일은 거의 대부분 **추론**이다. 이미 학습이 완료된 LLM(GPT, Llama 등)을 가져다가, 우리 데이터를 넣어서 답변을 생성하게 하는 것이다. 모델을 처음부터 학습시키는 것은 우리 범위가 아니다.

#### 학습은 어떻게 일어나는가? (직관적 이해)

1. 모델이 문제를 본다 (예: "이 사진은 고양이일까 강아지일까?")
2. 답을 추측한다 (처음에는 거의 무작위)
3. 정답과 비교한다
4. 틀린 정도에 따라 내부 규칙을 조금씩 조정한다
5. 1~4를 수백만 번 반복한다

이걸 요리 비유로 돌아가면: 팬케이크를 수백 번 만들면서, 매번 맛을 보고, "밀가루를 조금 줄이자", "불을 좀 낮추자" 하면서 레시피를 점점 완성해가는 것이다.

### 2.3 신경망 기초

#### 비유: 회사 조직도

**신경망(Neural Network)**은 AI 모델의 가장 대표적인 구조다. 회사 조직에 비유해보자.

```
[입력층]          [은닉층 1]       [은닉층 2]       [출력층]
이메일 텍스트  →   키워드 감지   →   패턴 조합    →   스팸/정상
                  "무료", "클릭"     복합 판단
                  "긴급", "당첨"

인턴들이          중간 관리자가      부장이            최종 결정자가
원본 데이터를     기본 특징을       복잡한 패턴을      최종 판단을
전달한다          추출한다          종합한다           내린다
```

- **입력층**: 원본 데이터를 받는 곳 (인턴이 서류를 접수하는 것)
- **은닉층(Hidden Layer)**: 데이터에서 점점 더 복잡한 패턴을 추출하는 곳 (중간 관리자가 분석하는 것)
- **출력층**: 최종 결과를 내놓는 곳 (임원이 결정을 내리는 것)

#### "딥러닝"이란?

은닉층이 **많을수록**(= 조직이 깊을수록) "딥(deep)"하다고 한다. 그래서 **딥러닝(Deep Learning)** = 은닉층이 많은 신경망.

```
전통 신경망:  입력 → [은닉1] → 출력                  (얕다)
딥러닝:      입력 → [은닉1] → [은닉2] → ... → [은닉N] → 출력  (깊다)
```

은닉층이 많으면 더 복잡한 패턴을 학습할 수 있다. GPT-4 같은 LLM은 은닉층이 수십~수백 개다.

#### 왜 "신경"망인가?

인간의 뇌에서 뉴런(신경 세포)이 서로 연결되어 신호를 전달하는 방식에서 영감을 받았기 때문이다. 다만, 실제 뇌의 작동 방식과는 상당히 다르다. 이름 때문에 혼동하지 말자 — 현대 AI는 뇌를 모방하는 것이 아니라, **데이터에서 패턴을 찾는 통계적 도구**에 가깝다.

### 2.4 지도학습 vs 비지도학습

AI가 학습하는 방식은 크게 세 가지로 나뉜다.

#### 지도학습 (Supervised Learning)

**비유: 과외 선생님이 있는 공부**

```
문제: "이 이메일은 스팸인가?"
정답: "네, 스팸입니다" (사람이 미리 표시해둔 정답)
```

- 입력과 정답이 **쌍으로** 주어진다
- 모델은 "입력 → 정답" 관계를 학습한다
- 정답이 있으므로 학습이 비교적 확실하다

**실생활 예시**:
- 사진 속 동물 이름 맞추기 (사람이 "고양이", "강아지" 라벨을 붙여둔 데이터로 학습)
- 집값 예측 (과거 거래 데이터: 면적, 위치 → 가격)
- 스팸 필터 (사람이 "스팸"/"정상" 분류해둔 이메일로 학습)

#### 비지도학습 (Unsupervised Learning)

**비유: 정답 없이 혼자 패턴을 찾는 공부**

```
데이터: 고객 1000명의 구매 기록
정답: 없음
모델이 할 일: "비슷한 고객끼리 그룹으로 묶어라"
```

- 정답이 **없다** — 데이터만 던져준다
- 모델이 스스로 데이터 속의 구조나 패턴을 발견한다

**실생활 예시**:
- 고객 세그먼테이션 (비슷한 구매 패턴의 고객 그룹 발견)
- 이상 탐지 (정상 패턴에서 벗어난 거래 감지)
- 임베딩 학습 (텍스트 간의 의미적 유사성 발견) — ATHENA에서 사용!

#### 자기지도학습 (Self-supervised Learning)

**비유: 빈칸 채우기 문제를 스스로 만들어서 푸는 공부**

```
원래 문장: "오늘 날씨가 좋아서 공원에 갔다"
문제:      "오늘 [???]가 좋아서 [???]에 갔다"
모델이 할 일: 빈칸을 맞춰라
```

- 데이터 자체에서 문제와 정답을 **자동으로 만들어낸다**
- 사람이 라벨을 붙일 필요가 없어서 대규모 데이터에 적용 가능하다
- **LLM이 바로 이 방식으로 학습한다!**

GPT가 학습하는 방법은 본질적으로 이것이다: 인터넷의 텍스트를 읽으면서, "다음 단어가 뭘까?"를 수조 번 맞추는 연습을 한 것이다.

#### 한눈에 비교

| | 지도학습 | 비지도학습 | 자기지도학습 |
|---|---|---|---|
| **정답 필요** | 필요 (사람이 준비) | 불필요 | 불필요 (데이터에서 자동 생성) |
| **비유** | 과외 선생님과 공부 | 도서관에서 혼자 분류 | 빈칸 채우기 자습 |
| **대표 예시** | 스팸 분류, 번역 | 클러스터링, 이상 탐지 | GPT, BERT |
| **데이터 규모** | 보통 (라벨링 비용) | 큼 | 매우 큼 |

### 2.5 LLM은 어디에 위치하는가?

지금까지 배운 개념을 조합하면, LLM(Large Language Model)의 정체를 이해할 수 있다.

#### LLM의 정체

```
LLM = 매우 깊은 신경망 + 자기지도학습 + 거대한 텍스트 데이터
```

풀어서 말하면:

1. **구조**: 은닉층이 수십~수백 개인 깊은 신경망 (특히 **Transformer**라는 구조)
2. **학습 방법**: 자기지도학습 — 인터넷의 텍스트를 읽으며 "다음 단어 맞추기"를 수조 번 반복
3. **학습 데이터**: 인터넷의 거의 모든 텍스트 (책, 위키피디아, 뉴스, 코드, 논문 등)
4. **결과**: 어떤 질문이든 그럴듯한 다음 단어를 이어서 생성하는 능력

#### "다음 단어 예측기"라는 본질

LLM이 하는 일의 본질은 놀라울 정도로 단순하다:

```
입력: "대한민국의 수도는"
LLM의 내부: "다음에 올 확률이 가장 높은 단어는... '서울'이다"
출력: "서울"
```

이것을 단어 하나하나씩 반복하면 문장이 되고, 문단이 되고, 긴 답변이 된다. ChatGPT가 글자가 하나씩 나오는 이유가 바로 이것이다 — 실제로 한 토큰씩 생성하고 있기 때문이다.

#### 왜 이렇게 잘 작동하는가?

단순한 "다음 단어 맞추기"가 왜 이렇게 지능적으로 보일까?

- 다음 단어를 잘 맞추려면 **문맥, 문법, 논리, 세상 지식**을 모두 이해해야 한다
- 예: "물을 100도까지 가열하면 ___" 이 빈칸을 맞추려면 물의 끓는점을 알아야 한다
- 이런 "빈칸 채우기"를 **수조 개의 문장**에 대해 반복하면, 모델은 자연스럽게 방대한 지식과 추론 능력을 습득한다

#### LLM의 한계 — 왜 RAG가 필요한가?

LLM이 아무리 잘해도, 근본적인 한계가 있다:

| 한계 | 설명 | 예시 |
|------|------|------|
| **지식 커트오프** | 학습 데이터 이후의 정보를 모른다 | "2026년 3월 뉴스 알려줘" → 모른다 |
| **환각(Hallucination)** | 그럴듯하지만 틀린 정보를 자신 있게 말한다 | 존재하지 않는 논문을 인용한다 |
| **도메인 지식 부재** | 우리 팀의 회의록, 내부 문서를 모른다 | "지난 회의에서 뭘 결정했지?" → 모른다 |
| **출처 불명** | 답변의 근거가 어디서 왔는지 알 수 없다 | "이거 어디서 나온 정보야?" → 대답 못 한다 |

이 한계를 극복하는 방법이 바로 **RAG(Retrieval-Augmented Generation)**다 — 외부 지식을 검색해서 LLM에게 넣어주는 것.

## 3. ATHENA 프로젝트와의 연결

지금까지 배운 개념이 ATHENA에서 어떻게 쓰이는지 정리한다.

### ATHENA의 핵심 구조

```
사용자 질문: "A 기술을 도입한 이유가 뭐야?"
     ↓
[1단계: 검색 (Retrieval)]
  지식 그래프 + 벡터 DB에서 관련 문서/관계를 검색
     ↓
[2단계: 보강 (Augmentation)]
  검색 결과를 LLM 프롬프트에 컨텍스트로 삽입
     ↓
[3단계: 생성 (Generation)]
  LLM이 컨텍스트를 기반으로 답변 생성
     ↓
"A 기술은 B 문제를 해결하기 위해 도입되었습니다.
 근거: 3월 15일 회의록, 기술 검토 보고서"
```

### 각 개념이 ATHENA에서 쓰이는 위치

| 이 문서에서 배운 개념 | ATHENA에서의 역할 |
|---|---|
| **모델** | LLM(GPT, Llama)이 답변 생성 엔진 |
| **추론** | 사용자 질문마다 LLM에게 답변을 생성시키는 과정 |
| **신경망/딥러닝** | LLM과 임베딩 모델의 내부 구조 |
| **자기지도학습** | GPT, Llama 등이 사전 학습된 방식 |
| **비지도학습** | 임베딩 모델이 텍스트 유사도를 학습하는 방식 |
| **LLM의 한계** | RAG와 Knowledge Graph가 필요한 이유 |

### ATHENA만의 차별점: GraphRAG

일반 RAG는 "관련 문서를 검색"하는 수준이지만, ATHENA는 **Knowledge Graph**를 활용한다.

- 일반 RAG: "A와 관련된 문서 3개를 찾아서 LLM에 전달"
- ATHENA (GraphRAG): "A → B → C로 이어지는 **인과 관계 사슬**을 추출해서 LLM에 전달"

이것이 ATHENA가 단순한 검색이 아니라 "왜?" 질문에 대해 **인과 사슬로 응답**할 수 있는 이유다.
